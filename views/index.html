API States

This project focuses on building a backend REST API using Node.js, Express, and MongoDB. A frontend was not implemented. All interaction with the API is done via HTTP requests.

Note: GET requests can be tested in the browser address bar. POST, PATCH, and DELETE requests require an API client such as Postman.

GET Accepted Requests

:state must be a valid US state abbreviation

  • GET /states
  • GET /states?contig=true
  • GET /states?contig=false
  • GET /states/:state
  • GET /states/:state/funfact
  • GET /states/:state/capital
  • GET /states/:state/nickname
  • GET /states/:state/population
  • GET /states/:state/admission

POST Requests

Add one or more fun facts for a state.

  • POST /states/:state/funfact

Example Request Body:

{
  "funfacts": [
    "Texas was once its own country.",
    "Dr Pepper was invented in Texas."
  ]
}

PATCH Requests

Update an existing fun fact by index (1-based).

  • PATCH /states/:state/funfact

Example Request Body:

{
  "index": 1,
  "funfact": "Texas is larger than France."
}

DELETE Requests

Delete a fun fact by index (1-based).

  • DELETE /states/:state/funfact

Example Request Body:

{
  "index": 2
}

All POST, PATCH, and DELETE requests must include the header: Content-Type: application/json.
